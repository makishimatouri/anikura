# Changelog

本项目从 `v0.1.0` 开始按正式版本留档。所有上线版本都应有 Git tag，并在本文件记录用户可见变化。

## v0.1.54 - 2026-07-09

移动端首页首屏文案修正。

- 移动端首页第一页介绍文案改为与桌面端一致

## v0.1.53 - 2026-07-09

移动端首页首屏文案修正。

- 将移动端首页第一页介绍文案改为“我们是全国最大的以ankr活动为核心的聚合信息站。”

## v0.1.52 - 2026-07-09

移动端首页首屏标题样式修正。

- 移动端首页第一页 `ANIKURA CN` 改为紫粉渐变字

## v0.1.51 - 2026-07-09

移动端首页首屏文案修正。

- 移动端首页第一页大标题改为 `ANIKURA CN`
- 移除第一页“小标题 + 动漫歌曲 club 活动入口”的重复表达

## v0.1.5 - 2026-07-09

移动端首页分屏入口。

- 移动端首页改为整屏下滑切换的板块入口
- 新增首页、精选推荐、全国活动、AniROX、签到积分、通知联系六个移动端入口页
- 下滑切换时加入板块淡入和位移动效，并保留右侧进度指示
- 桌面端首页结构保持不变

## v0.1.44 - 2026-07-09

精选推荐命名修正。

- 将首页和超管后台的“随机推荐”文案改回“精选推荐”
- 推荐筛选和抽取逻辑保持不变

## v0.1.43 - 2026-07-09

随机推荐后台控制修正。

- 首页随机推荐只从超管勾选的活动中抽取
- 超管活动表单新增“推送至随机推荐”文案
- 超管活动列表的快速操作改为“推送随机推荐 / 取消随机推荐”
- 更新头图说明，明确随机推荐使用活动主海报

## v0.1.42 - 2026-07-09

首页随机推荐修正。

- 首页“精选推荐”改为“随机推荐”
- 推荐活动从所有已审核、进行中的活动里随机抽取
- 推荐卡片只展示活动主海报，不再显示标题、标签、时间地点等信息
- 新增“换一个”按钮，用于随机刷新推荐活动
- 首页下方活动列表固定为“近期活动”，避免重复精选概念

## v0.1.41 - 2026-07-09

精选轮播展示修正。

- 精选活动卡片图片区不再叠加标题、标签和时间地点
- 活动信息移动到图片下方，避免遮挡海报或头图

## v0.1.4 - 2026-07-09

移动端轮播操作修复。

- 首页精选推荐轮播支持手机左右滑动切换
- 滑动真实活动卡片时不会误跳转到活动详情页
- 保留竖向滚动手势，不影响页面正常上下滑动

## v0.1.3 - 2026-07-09

活动图片展示优化。

- 新增活动头图字段，用于首页轮播和活动卡片横向展示
- 后台新增可选头图上传入口，并标注推荐比例
- 未上传头图时继续自动裁剪海报作为横向展示图
- 活动详情页改为展示完整海报原图，不再 16:9 裁剪
- 新增 Supabase 数据库迁移 SQL

## v0.1.2 - 2026-07-09

后台精选管理修复。

- 超管活动管理列表新增“设为精选 / 取消精选”快速操作
- 修复服务端页面识别超管身份不稳定，导致编辑页不显示精选开关的问题
- 活动表单新增“超管运营设置”说明，标明首页轮播展示条件
- 编辑活动时不再覆盖原活动创建者，避免影响审核通知归属

## v0.1.1 - 2026-07-09

仓库规范化整理。

- 重写 README，使仓库首页更像正式项目
- 新增架构、部署、版本管理和维护文档
- 新增 CI 检查流程和 PR 模板
- 移除会制造无意义提交的 weekly backup workflow
- 补齐 ESLint 9 配置，并修复现有 lint error
- 清理 `.env.local.example`，避免出现疑似真实密钥的占位符
- 保持线上功能逻辑不变

## v0.1.0 - 2026-07-09

基线版本归档。

- 上线 Anikura CN 网站基础功能
- 支持全国活动浏览、筛选、详情页和 QQ 加群
- 支持 AniROX 活动专区、联系页、关于页和友情链接占位
- 支持管理员登录、活动录入、编辑、审核和通知
- 支持首页精选轮播
- 支持用户注册、每日签到、积分、优惠券兑换和门票抽奖入口
- 支持 Supabase 后端、Vercel 部署、Cloudflare 自定义域名
